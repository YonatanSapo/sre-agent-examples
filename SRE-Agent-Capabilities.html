How It Works

The Deterministic 8-Step Investigation Protocol

To guarantee the agent doesn’t hallucinate findings or skip critical diagnostic phases, its behavior is governed by a strictly sequenced protocol. It didn’t just run queries — it systematically encoded the exact method a senior engineer uses to diagnose complex systems, culminating in a structured investigation report.

1

Anchor the Timestamp

MANDATORY first step. Calculate absolute start/end times (±1h) to prevent temporal drift. Extract the alert’s exact UTC time, compute query boundaries, and state the window explicitly. All subsequent queries use these absolute timestamps — never relative time windows that silently drift.

2

Historical Synthesis

Query incident tracking and knowledge base systems for similar past incidents or runbooks to establish a known baseline. If a matching pattern is found, reference immediately — skip redundant investigation.

3

Triage & Blast Radius

Parse the alert and scope impact across tenants, sessions, services, and regions. Quantify the blast radius with data-driven metrics: who is affected, how many, and how severely. Determines escalation level — from “negligible” to “critical.”

4

System Pulse Check

Scan core metrics (CPU, memory, errors, latency) for correlated anomalies. Identify whether the signal is isolated or part of a broader systemic pattern across the infrastructure.

5

Log & Error Analysis

Aggregate and classify errors to map the failure surface across services and pods. Top-errors ranking, pod distribution, multi-service sweep — building the full error landscape.

6

Trace Deep-Dive

Follow the failing request via distributed traces to isolate the exact point of failure. Cross-service span hierarchies, slow span identification, and request flow visualization.

7

Code & Deployment Correlation

Link the issue to source code and recent deployments to identify the root cause. Locate the source file from stack traces, compare error rates before vs. after deploys, and surface the exact change that broke it.

8

Adversarial Verification

The mandatory adversarial gate. Re-validate all assumptions — time window, region, environment, queries. Does the root cause timestamp align with the alert? Does the evidence actually support the conclusion? Any mismatch invalidates the result. Born from a real misdiagnosis where a wrong time window produced a plausible but incorrect answer. A single CRITICAL failure invalidates the entire investigation.
